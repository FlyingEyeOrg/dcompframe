# DCompFrame UI 控件设计文档（控件规范）

## 1. 文档说明

本文档定义常用控件的统一规范。每个控件均包含以下四部分：

- 设计概要
- 设计要求
- 功能标准
- 验收标准

适用控件范围：

- TextBox
- ComboBox
- Progress
- Loading
- TabControl
- Grid
- StackPanel
- Panel
- Label
- Popup
- Expander
- ItemsControl

参考标准来源：WinUI、WPF、WAI-ARIA APG、Material 3（见框架总规范）。

## 2. 控件规范

## 2.1 TextBox

### 设计概要

用于单行或多行文本输入，支持光标、选区、编辑命令与输入法。

### 设计要求

- 支持基本编辑：输入、删除、退格、替换选区。
- 支持光标移动：Left/Right/Home/End，多行时支持 Up/Down。
- 支持快捷键：Ctrl+A/C/X/V/Z/Y（按平台能力逐步实现）。
- 光标与选区样式全局统一，禁止同项目内控件自定义不同宽度。
- 获取焦点后可接收字符输入，失焦后停止接收。

### 功能标准

- 文本输入延迟不可感知（典型场景 < 16ms 更新）。
- 输入法组合输入不破坏已有文本。
- 文本超出可见区域时自动滚动保证光标可见。

### 验收标准

- 在空文本框输入 1000 字符无丢字。
- 连续执行 500 次“输入+退格”后内容与预期一致。
- 光标宽度在不同主题/缩放下保持一致。
- 鼠标点击定位与键盘方向键定位结果一致。

## 2.2 ComboBox

### 设计概要

提供单选列表，支持选择态展示与弹出列表交互（可选支持可编辑模式）。

### 设计要求

- 折叠态显示当前值。
- 展开态弹层独立于主布局，不可挤压后续控件。
- Esc 关闭弹层并恢复焦点到触发控件。
- 滚轮优先作用于下拉列表，禁止穿透滚动到外层 ScrollViewer。
- 键盘支持：Down/Up 导航，Enter 选择，Esc 取消。

### 功能标准

- 展开与收起状态切换可预测。
- 列表支持滚动、hover、选中高亮。
- 列表项选中后同步更新显示值并关闭弹层。

### 验收标准

- 弹层打开后滚轮 2000 次，外层容器滚动位置不变化。
- 键盘选择与鼠标选择结果一致。
- 弹层不被同级控件遮挡。

## 2.3 Progress

### 设计概要

用于展示任务完成度，可为确定型（0-100）或不确定型（Indeterminate）。

### 设计要求

- 确定型支持最小值、最大值、当前值。
- 不确定型支持连续动画反馈。
- 值变化必须平滑过渡（可配置动画）。

### 功能标准

- 值域约束：超范围自动 clamp。
- 显示值与内部值一致（四舍五入策略明确）。

### 验收标准

- 设置非法值（<min 或 >max）后显示结果符合 clamp 规则。
- 连续更新 10,000 次无卡顿和崩溃。

## 2.4 Loading

### 设计概要

表示系统正在处理，强调“进行中”状态而非具体进度。

### 设计要求

- 支持独立组件和覆盖层两种模式。
- 覆盖层模式应阻止底层误操作（可配置是否拦截输入）。
- 动画循环稳定，不因窗口失焦导致状态错乱。

### 功能标准

- 启停行为可重入（重复 start/stop 不异常）。
- 可与异步任务生命周期绑定。

### 验收标准

- 连续 1000 次 start/stop 无崩溃、无残留渲染对象。
- 覆盖层开启时底层按钮不可点击（拦截模式）。

## 2.5 TabControl

### 设计概要

用于在多个内容面板间切换，单次显示一个面板。

### 设计要求

- Tab 头与内容区明确分层。
- 支持手动激活（Enter/Space）和自动激活（可配置）。
- 键盘支持：Left/Right（或 Up/Down 视方向）、Home/End。
- Tab 切换后焦点行为可预测。

### 功能标准

- 激活态样式唯一，非激活态无歧义。
- 内容切换不应导致未预期滚动跳变。

### 验收标准

- 快速切换 5000 次无状态错乱。
- 键盘导航顺序与视觉顺序一致。

## 2.6 Grid

### 设计概要

二维网格布局容器，按行列定义子项位置和拉伸策略。

### 设计要求

- 支持固定、自动、比例（star）尺寸策略。
- 行列测量与排列分离，避免循环依赖。
- 子元素越界时按裁剪策略处理。

### 功能标准

- 同一网格内子项不会互相覆盖（除显式允许的 overlay）。
- 缩放窗口时布局稳定，无抖动。

### 验收标准

- 动态调整窗口大小 2000 次，布局无重叠错误。
- 复杂嵌套网格下排列结果与约束一致。

## 2.7 StackPanel

### 设计概要

线性布局容器，按方向顺序排列子元素。

### 设计要求

- 支持垂直/水平方向。
- 主轴顺序稳定，交叉轴拉伸策略可配置。
- 当内容超出时可与滚动容器协同。

### 功能标准

- 子元素排列顺序与添加顺序一致。
- 可正确计算内容总长度供外层滚动使用。

### 验收标准

- 1000 个子元素排列无错序。
- 在 ScrollViewer 中滚动无遮挡、无闪烁。

## 2.8 Panel

### 设计概要

基础容器，提供最小布局职责，通常作为其他布局容器基类。

### 设计要求

- 默认行为清晰（拉伸填充或透明容器，必须固定规范）。
- 必须对子元素命中与裁剪边界负责。

### 功能标准

- 子元素生命周期与父容器一致。
- 不额外引入副作用布局。

### 验收标准

- 父容器变更尺寸后子元素行为符合默认规则。
- 不出现子元素不可见但可点击的异常区域。

## 2.9 Label

### 设计概要

用于静态文本说明，可与输入控件建立语义关联。

### 设计要求

- 默认不可聚焦。
- 支持文本截断、换行、对齐。
- 与目标输入控件可建立“标签-控件”关系。

### 功能标准

- 长文本策略明确（ellipsis 或 wrap）。
- 高 DPI 下文本清晰，不重影。

### 验收标准

- 不同字体大小下基线与相邻控件对齐。
- 文本截断与 tooltip 行为一致。

## 2.10 Popup

### 设计概要

临时浮层容器，用于菜单、提示、下拉、对话框等。

### 设计要求

- 必须拥有独立层级（高于常规内容层）。
- 支持点击外部关闭（可配置）。
- 支持焦点管理策略（模态/非模态）。
- 模态 Popup 需阻断背景交互。

### 功能标准

- 打开时焦点进入 Popup，关闭后焦点返回触发源。
- Esc 关闭可用。

### 验收标准

- 连续打开关闭 5000 次无泄漏。
- 多层 Popup 嵌套时层级和焦点行为正确。

## 2.11 Expander

### 设计概要

带标题头的可展开容器，用于渐进披露内容。

### 设计要求

- 提供 Expanded/Collapsed 两种状态。
- 展开时可见内容参与布局；收起时不占用内容区。
- 支持键盘触发（Enter/Space）。

### 功能标准

- 动画展开收起可选，状态切换事件可订阅。
- 嵌套 Expander 行为稳定。

### 验收标准

- 连续切换 3000 次无状态错乱。
- 收起状态下子内容不可命中。

## 2.12 ItemsControl

### 设计概要

用于展示数据项集合，负责项生成、容器管理与布局协同。

### 设计要求

- 支持数据源绑定与动态增删。
- 支持 ItemTemplate（或等价渲染委托）。
- 与 ScrollViewer 协同时必须明确滚动归属与可见区。
- 支持虚拟化（分阶段：先可见区裁剪，后完整虚拟化）。

### 功能标准

- 大数据集下滚动流畅，无明显掉帧。
- 数据变更后 UI 与数据一致。

### 验收标准

- 10,000 项数据下滚动与选择稳定。
- 不出现“被 ScrollViewer 遮挡”的布局错误。
- 鼠标滚轮仅影响当前命中滚动区域。

## 3. 统一交互标准（跨控件）

- 焦点：
  - 可交互控件有焦点环。
  - 非交互控件默认不参与 Tab。
- 滚轮：
  - 命中优先，先子后父，已消费事件不得继续冒泡。
- 键盘：
  - Enter/Space 激活。
  - Esc 关闭临时 UI。
  - Home/End/PageUp/PageDown 遵循控件语义。
- 可访问性：
  - 控件必须可命名、可聚焦（交互控件）、可描述状态。

## 4. 最小测试矩阵（必须覆盖）

- 正常流程：创建、显示、交互、销毁。
- 边界情况：空数据、超长文本、最小/最大值、超小窗口。
- 异常场景：高频输入、快速切换焦点、反复打开/关闭弹层。
- 回归场景：
  - TextBox 输入链路
  - 光标一致性
  - Slider 拖拽稳定性
  - ComboBox 滚轮穿透
  - ScrollViewer 与 ItemsControl 遮挡

## 5. 执行约束

- 后续控件开发必须逐条对照本文档。
- 未满足本规范验收标准的功能不得标记完成。
- 新增控件时必须补充同结构条目（设计概要、设计要求、功能标准、验收标准）。
